Import dei clienti, sono supportati solo i seguenti formati: .xls, .xlsx

Se l'utente è già presente a parità di e-mail gli altri dati vengono aggiornati, la password non viene modificata.
La prima riga del file viene considerata come intestazioni, i nomi delle colonne (anche non in ordine) da utilizzare:
  • LINGUA, lingua di riferimento dell'utente, se non presente viene preso l'italiano
  • E_MAIL, e-mail dell'utente, campo obbligatorio
  • NOME, nome dell'utente
  • COGNOME, cognome dell'utente
  • PASSWORD, password assegnata all'utente, viene utilizzata solo se l'utente NON è già presente in arcivhio, al primo accesso da parte dell'utente viene richiesta la modifica della password, se non presente per i nuovi utenti per default viene inserita: Password1!
  • GRUPPO, elenco dei gruppi da abbinare, separare con virgola i nomi, se un gruppo non è presente viene creato, i gruppi associati si aggiungono a quelli già presenti
  • SESSO, sesso dell'utente M per machio, F per femmina
  • ANNO_NASCITA, anno di nascita
  • DATA_NASCITA, data di nascita nel formato gg/mm/aaaa
  • B2B, impostare "si" se cliente di tipo B2B
